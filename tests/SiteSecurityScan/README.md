# Site Security Scan Test Framework

Compares output from PowerShell SharePointPermissionScanner with V2 Middleware security_scan API.

## Folder Structure

```
tests/SiteSecurityScan/
├── 01_RunAll.ps1                    # Orchestrator - runs all steps
├── 02_SetupSharePointPermissionScanner.ps1  # Creates input CSV from .env
├── 03_RunPowerShellScanner.ps1      # Runs PowerShell scanner
├── 04_RunV2SecurityScan.ps1         # Calls V2 API, extracts report
├── 05_CompareOutputs.ps1            # Compares both outputs
├── SharePointPermissionScanner/     # PowerShell scanner scripts + output
│   ├── SharePointPermissionScanner.ps1
│   ├── _includes.ps1
│   ├── SharePointPermissionScanner-In.csv  # Generated by setup
│   └── *.csv                        # Output files (01-05)
└── SiteSecurityScanOutput/          # Output from V2 API endpoint
```

## Usage

### Run Full Test

```powershell
cd tests\SiteSecurityScan
.\01_RunAll.ps1
```

**Note:** PowerShell scanner requires browser authentication.

### Run Individual Steps

```powershell
# Setup only
.\02_SetupSharePointPermissionScanner.ps1

# PowerShell scanner only
.\03_RunPowerShellScanner.ps1

# V2 API scan only (starts server if needed)
.\04_RunV2SecurityScan.ps1

# Compare existing outputs
.\05_CompareOutputs.ps1
```

### Skip Steps

```powershell
# Skip PowerShell scan (use existing output)
.\01_RunAll.ps1 -SkipPowerShellScan

# Skip V2 scan (use existing output)
.\01_RunAll.ps1 -SkipV2Scan

# Only run comparison
.\01_RunAll.ps1 -SkipPowerShellScan -SkipV2Scan
```

## Configuration

Site URL is read from `.env` file:
```
CRAWLER_SELFTEST_SHAREPOINT_SITE=https://whizzyapps.sharepoint.com/sites/AiSearchTest01
```

Or override with parameter:
```powershell
.\02_SetupSharePointPermissionScanner.ps1 -SiteUrl "https://tenant.sharepoint.com/sites/MySite"
```

## Output Files Compared

| File | Description |
|------|-------------|
| 01_SiteContents.csv | Lists and libraries in site |
| 02_SiteGroups.csv | SharePoint groups |
| 03_SiteUsers.csv | Users with site access |
| 04_IndividualPermissionItems.csv | Items with broken inheritance |
| 05_IndividualPermissionItemAccess.csv | Individual item permissions |

## Known Differences

- **Column structure**: PowerShell scanner includes Job/SiteUrl columns, V2 does not
- **URL format**: PowerShell uses full URLs, V2 uses relative paths
- **Type values**: PowerShell uses "File/Folder/Item", V2 uses "ITEM"
- **Subsites**: PowerShell scans subsites by default, V2 depends on `include_subsites` parameter
